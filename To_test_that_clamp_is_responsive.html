To test that clamp is responsive




This paragraph is 16 pixels.

This paragraph is 1 rem.

This paragraph is 1 vw.

This h1 heading is 16 pixels.

This h1 heading is 1 rem.

This heading is 1 vw.

This heading is 2 vw.

This heading is 3 vw.

Comparing vw to px and rems:

My viewport width on my main screen is 1680px.
With 1 rem=16 px i.e. without resizing the html font to 62.5%, my viewport is 1680px = 105rem.
1vw is 1% of that, so 1.05rem, which is 16.8px.
1vw is about the same thing as 1 rem. Check it out:

This paragraph is 1 vw.

This paragraph is 16.8 pixels.

This paragraph is 1.05 rem.

This paragraph is 1 rem.

So without resizing the html font to 62.5%, seeing things on my large screen, 1vw is a little larger than 1 rem.

This h1 heading is defined with a clamp funtion: clamp(1rem,2vw,3rem)



This h1 heading is 1rem, the min value.

This h1 heading is 2vw, the preferred value.

This h1 heading is 3rem, the max value.




Look at the title with the clamp function. Notice that:
- at full screen, it is the size of the preferred value.
This is as expected: 2vw at full screen is smaller than 3rem, so the browser can go with the preferred value rather than the max one.
- when you decrease the window's size:
This is as expected.
 - at first it resizes just as the heading with the preferred value.
 - at some point it stops resizing: it remains the size of the min value heading (whereas the preferred value heading in vw keeps resizing).

This h1 heading is defined with a clamp funtion: clamp(1rem,3vw,2rem)



This h1 heading is 1rem, the min value.

This h1 heading is 3vw, the preferred value.

This h1 heading is 2rem, the max value.




Look at the title with the clamp function. Notice that:

- at full screen:
    - it is not the size of the preferred value.
This much is as expected: 3vw at full screen is smaller than 2rem, so we can expect that the browser won't display the preferred value.
    - IT IS LARGER THAN THE MAX VALUE.
THAT is somewhat suprising. You'd expect the max value to be just that, and constrain the heading to no go larger than it is set.
The issue must be that at full-desktop-screen, the max value here is smaller than the preferred value. So, at least it should not be a problem in practice.

- when you decrease the window's size:
This is as expected.
    - at first it resizes just as the heading with the preferred value.
    - at some point it stops resizing: it remains the size of the min value heading (whereas the preferred value heading in vw keeps resizing).

This heading is defined with a clamp funtion. clamp(2rem,5vw,8rem)
